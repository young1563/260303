# 🎮 게임 장르별 초반 유저 이탈 방지 전략 가이드

이 보고서는 다양한 게임 장르에서 유저가 초기에 이탈하는 원인을 분석하고, 이를 방지하기 위한 핵심 기획 요소를 **가장 이해하기 쉬운 형태**로 정리한 가이드입니다.

---

## 📌 1. 핵심 요약 (Essential Summary)

초반 이탈을 막는 핵심은 **"마찰은 줄이고, 보상은 빠르게"** 주는 것입니다.

1.  **TTFR & TTFF 단축**: 첫 보상(Reward)과 첫 재미(Fun)를 느끼는 시간을 최소화하세요.
2.  **맥락 중심 튜토리얼**: 텍스트로 가르치지 말고, 실제 행동(Action)을 통해 배우게 하세요.
3.  **기술 품질은 기본 중의 기본**: 로딩 지연, 크래시, 발열은 어떤 장르에서도 치명적인 이탈 원인입니다.
4.  **신뢰 중심의 수익화**: 유저와 신뢰가 쌓이기 전(초반)에 과도한 결제 유도는 독이 됩니다.

---

## 📊 2. 장르별 핵심 이탈 방지 요소 (Genre Strategy)

| 장르 | 핵심 목표 | 이탈 방지 필살기 |
| :--- | :--- | :--- |
| **모바일 캐주얼** | 쉬운 이해 + 즉각 승리 | 첫 60초 내 '첫 승리' 경험 제공 / 난이도 완급 조절 |
| **하이퍼 캐주얼** | 즉시 플레이 (Instant) | 튜토리얼 제거 혹은 5초 내 종료 / 원터치 리스타트 |
| **RPG / 가챠** | 성장 루프 인지 + 신뢰 | 첫 1시간 내 뽑기 재미 제공 / 확률 및 정보의 투명성 |
| **액션 / 어드벤처** | 손맛(조작) + 몰입 | 첫 2시간(환불 방어선) 내 핵심 액션 반복 및 세계관 전달 |
| **MMO / 소셜** | 관계 형성 (Social) | 7일 이내 길드 가입 또는 친구 연결 유도 |
| **전략 / 시뮬레이션** | 복잡도 관리 | 기능을 한 번에 열지 않고 단계적(Step-by-Step) 노출 |
| **퍼즐 / 보드** | 공정한 실패 인식 | 실패 시 '운'이 아닌 '실력 부족'으로 납득 가능한 피드백 |

## ⏰ 3. 시간대별 유저 심리와 공략 포인트

### 🕒 [첫 1시간] 가치 인지 (The First Impression)
*   **유저 심리**: "이 게임 할 만한가? 내가 할 수 있는 건가?"
*   **공략**: 인지 부하를 줄이세요. 텍스트 폭탄을 피하고, 즉시 '성공'의 쾌감을 느끼게 합니다.
*   **지표**: 튜토리얼 완료율, 첫 보상 획득 시간(TTFR).

### 📅 [첫 1일] 목표 형성 (Retention Trigger)
*   **유저 심리**: "내일 다시 접속할 이유가 있을까?"
*   **공략**: 명확한 '내일의 보상'과 '성장 목표'를 제시하세요. 푸시 알림의 명분이 필요합니다.
*   **지표**: D1 리텐션, 재방문 트리거 클릭률.

### 🗓️ [첫 7일] 습관화 (The Routine)
*   **유저 심리**: "이 게임은 내 일상의 일부가 될 수 있을까?"
*   **공략**: 소셜 관계(길드, 친구)를 만들고 주기적인 콘텐츠 사이클(주간 미션 등)에 안착시킵니다.
*   **지표**: D7 리텐션, 길드/친구 도달률.

## 🛠️ 4. 실전 이탈 방지 기획 체크리스트

### ✅ 시스템 (System)
- [ ] 첫 보상(TTFR)이 1분 이내에 제공되는가?
- [ ] 초반 10스테이지(또는 레벨)까지 실패 스트레스가 낮은가?
- [ ] 복귀 유저를 위한 전용 혜택과 가이드가 있는가?

### ✅ 콘텐츠 (Content)
- [ ] 튜토리얼이 '설명'이 아닌 '경험'으로 설계되어 있는가?
- [ ] 난이도가 급상승하는 '벽' 구간이 너무 빨리 오지는 않는가?
- [ ] 게임의 핵심 재미(Core Loop)가 30분 이내에 최소 2회 반복되는가?

### ✅ UI / UX
- [ ] 화면에 '지금 해야 할 행동'이 하나만 명확히 보이는가? (CTA 최적화)
- [ ] 로딩 화면에서 유용한 팁이나 볼거리가 제공되는가?
- [ ] 확률형 아이템 정보가 유저가 찾기 쉬운 곳에 투명하게 공개되어 있는가?

## 🚀 5. 성과 측정을 위한 A/B 테스트 예시

*   **가설**: "튜토리얼 단계를 3개에서 1개로 줄이면 첫날 잔존율(D1)이 상승할 것이다."
*   **실행**:
    *   **Group A**: 기존 3단계 튜토리얼 (30초 소요)
    *   **Group B**: 즉시 게임 시작 + 위기 시 힌트 노출 (10초 소요)
*   **측정**: 어떤 그룹의 유저가 더 많이 남고, 더 많이 게임을 계속하는지 데이터로 확인 후 최종 적용.

---
> **Tip**: 모든 장르에서 공통적으로 **"기술적 문제(끊김, 튕김)"**는 어떤 화려한 기획보다 강력한 이탈 원인이 됩니다. 기술 최적화는 리텐션의 기초입니다.
